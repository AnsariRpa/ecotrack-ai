This document describes the AI tools, development environments, and frameworks used to build EcoTrack AI, including the selection rationale and business value for each.

## AI Tools

### 1. Google Antigravity IDE (Primary Development Agent)

**Role:** Principal development orchestrator  
**Usage:** 100% of code generation, architecture design, and implementation

Antigravity served as the primary engineering force for EcoTrack AI, performing:
- **Full-stack scaffolding** — Express + Prisma backend, React + Vite frontend
- **Architecture design** — dual-database abstraction, auth middleware, service layers
- **Google Cloud integration** — Gemini SDK prompts, Firebase Admin setup, Cloud Run Dockerfile
- **Competition optimisation** — scoring criteria analysis and gap remediation
- **Test suite authorship** — Vitest unit tests for calculation engine, AI coach, and React components
- **Documentation generation** — all `/competition` docs and `README.md`

**Value:** Eliminated weeks of boilerplate setup and enabled a production-grade application to be built and refined iteratively against competition criteria.

---

### 2. Google Gemini 1.5 Pro (Production AI Feature)

**Role:** AI Carbon Coach brain + Carbon Forecasting engine  
**Usage:** Runtime personalised recommendations and emission projections

Gemini is not just used as a development tool — it is an **embedded product feature**:
- Every coaching recommendation is generated by Gemini analysing real user activity data
- Every forecast is produced by Gemini performing time-series trend analysis on 30-day history
- Prompt engineering was iteratively refined to produce consistent, schema-valid JSON output

**Value:** Transforms a static rule database into a personalised, adaptive coaching system that improves with more user data.

## Selection Rationale

### Why Express over Next.js?
Express gives complete control over the API layer, middleware ordering, and request lifecycle — critical for implementing auth middleware with exact fallback semantics.

### Why Prisma over raw SQL?
Prisma provides type-safe database access with a declarative schema, making it easy to add fields and maintain integrity. Its migration system supports both SQLite (local dev) and PostgreSQL (production).

### Why Vitest over Jest?
Vitest shares Vite's configuration, producing faster test runs and native TypeScript support without transpilation setup. It is the natural choice for Vite-based projects.

### Why Recharts over Chart.js or D3?
Recharts is React-native, declarative, and provides accessible SVG charts out of the box with no imperative DOM manipulation — ideal for an accessibility-focused application.

### Why a dual-DB abstraction approach?
Rather than forcing judges to provision a PostgreSQL server, the app uses SQLite for local runs while the codebase demonstrates Firestore production integration via the `geminiService` and `auth` service patterns. This balances demo-ability with production credibility.

## Prompt Engineering Approach

### Recommendation Prompts

The AI coaching prompt was engineered through the following iterations:

**v1 (baseline):** Simple free-text request for sustainability tips.  
*Problem:* Inconsistent format, missing quantitative estimates.

**v2 (structured):** Added JSON schema specification.  
*Problem:* Occasional markdown wrapping in response, schema deviations.

**v3 (current):** Added explicit JSON code block instruction, field constraints, and rules section:
- `potentialSaving` must be a number
- `confidence` restricted to enum values
- `actionableSteps` limited to 3 items achievable in 7 days
- JSON extracted with regex fallback to handle `\`\`\`json` wrapper

**Result:** Deterministic, schema-valid JSON output suitable for direct database persistence.

### Forecast Prompts

Forecast prompts explicitly inject:
1. Historical time-series data (last 14 days)
2. Computed average daily emission
3. User goal target (if set)
4. Required output format with exact field names

This grounds Gemini's statistical reasoning in real data, preventing hallucinated baselines.
