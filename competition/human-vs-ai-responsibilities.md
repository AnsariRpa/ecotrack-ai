# Human vs AI Responsibilities

## Overview

This document defines the division of responsibilities between the **human participant** and **AI systems** in the development of EcoTrack AI. It provides transparency into what was created by AI, what was directed by human intent, and how the collaboration produced a coherent, competition-ready product.

---

## Responsibility Matrix

| Domain | Human | AI (Antigravity + Gemini) |
|--------|-------|--------------------------|
| **Product Vision** | ✅ Defined problem: individual carbon awareness | 🤖 Refined feature scope based on criteria |
| **Competition Strategy** | ✅ Identified evaluation framework | 🤖 Mapped scoring gaps → implementation priorities |
| **Architecture Design** | ✅ Approved dual-DB / fallback strategy | 🤖 Designed service abstraction layers |
| **Code Implementation** | ❌ Minimal manual edits | ✅ 100% of source code |
| **Google Services Selection** | ✅ Directed Gemini + Firebase requirement | 🤖 Integrated SDKs and designed APIs |
| **Prompt Engineering** | ✅ Reviewed recommendation quality | 🤖 Iterated prompt structure for consistent JSON |
| **Testing** | ✅ Approved test strategy | 🤖 Authored all test suites |
| **Documentation** | ✅ Defined required documents | 🤖 Authored all content |
| **Deployment Config** | ✅ Cloud Run + Docker approved | 🤖 Dockerfile, cloudbuild.yaml, docker-compose |
| **Security Review** | ✅ Approved auth strategy | 🤖 Implemented auth middleware, input validation |
| **Accessibility** | ✅ WCAG 2.1 requirement set | 🤖 Implemented ARIA, skip links, semantic HTML |

## Human Contributions

### 1. Problem Definition
The human participant identified the problem domain — personal carbon footprint tracking — and defined the core user journey: log activities → get AI coaching → set goals → track progress.

### 2. Competition Alignment
The human analysed the Prompt Wars evaluation framework and specified which scoring dimensions to prioritise (Code Quality, Google Services, Problem Alignment, Security).

### 3. Feature Prioritisation
The human directed which upgrades were highest priority:
- Real Gemini AI integration (not mock data)
- Explainable AI output (not just tip text)
- Carbon forecasting (7-day and 30-day projections)
- Firebase authentication implementation

### 4. Quality Gates
The human reviewed application state at multiple checkpoints:
- Tested the running application in browser
- Approved the implementation plan before code was written
- Directed the "proceed" signal at each major phase

### 5. Environment Provision
The human set up the development environment, approved npm package installations, and provided the Google Cloud API key configuration context.

---

## AI Contributions (Antigravity IDE)

### 1. Full Codebase Authorship
Every source file — TypeScript, React, CSS, Prisma schema, Dockerfile, YAML configs — was generated by the Antigravity agent.

### 2. Architecture Design
The dual-database abstraction pattern, auth middleware with graceful fallback, and Gemini service with rule-based fallback were all designed and implemented by the AI.

### 3. Gemini Prompt Engineering
The Gemini integration prompts were iteratively designed by the AI, including:
- Structured JSON output enforcement
- Data-grounded context injection
- Regex-based JSON extraction fallback

### 4. Bug Resolution
TypeScript compilation errors (unused imports, ESM dynamic import semantics for firebase-admin) were independently diagnosed and resolved by the AI.

### 5. Competition Documentation
All documents in the `/competition` directory were authored by the AI based on the human-defined list of required files.

---

## Collaboration Model

The human-AI collaboration followed an **architect-contractor model**:

1. **Human acts as the Architect** — defining intent, scope, priorities, and quality gates
2. **AI acts as the Senior Engineering team** — implementing, debugging, optimising, and documenting

This model maximises the productivity of both parties: the human provides direction and judgment; the AI provides execution velocity and technical depth.

---

## Reflection on AI-Assisted Development

**What worked exceptionally well:**
- The AI could maintain architectural consistency across 20+ files simultaneously
- The graceful fallback design emerged organically from the AI's understanding of competition judging needs
- The explainability layer (reasoning, behaviour insight, actionable steps) was richer than initially spec'd

**What required human judgment:**
- Deciding whether to use Firebase Auth or a simpler approach (human chose Firebase for competition credibility)
- Approving the implementation plan before execution began
- Reviewing screenshots of the running app to validate UI quality

**Key insight:**  
AI dramatically accelerates implementation but still requires strong human intent to produce work aligned with real-world goals. The quality of the output was directly proportional to the quality of the human's prompts and feedback.
